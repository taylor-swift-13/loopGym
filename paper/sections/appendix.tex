\section{Generation Prompt}
\label{app:prompt}

The closed-book generation prompt (abridged); the postcondition is stripped from the program before it is shown.

\begin{promptbox}[System prompt (abridged)]
\begin{lstlisting}[style=promptstyle]
You are given a C function containing a loop. Produce the strongest
inductive ACSL loop invariants that capture the loop's behavior:
- one core conservation / relational equality when one exists;
- minimal tight progress bounds (weaken strict guards to non-strict
  at exit);
- an exact loop assigns clause;
- no tautologies, no restatement of the guard alone;
- reference initial parameter values only via \at(v, Pre);
- allowed operators: == != < <= > >= && || ! ==> <==> + - * / %.
Output ONLY lines of the form:
  loop invariant <expr>;
\end{lstlisting}
\end{promptbox}

\section{Reward Service Interface}
\label{app:service}

The reward framework is packaged as a slim container (no prover, no GPU dependencies) exposing
\texttt{POST /reward} with body \texttt{\{program, rollouts[]\}} and response
\texttt{\{rollout\_rewards[], base[], marginal[], rollouts[] (with rejection detail), batch\_score, should\_reroll\}}.
Rollouts may be raw LLM text, invariant lists, or annotated code; parsing is uniform.
An offline scorer provides the same computation over JSONL/Parquet rollout dumps for trainer-side batching.
